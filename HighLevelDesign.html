1. Introduction
The Luminary is an educational platform aimed at optimizing the learning experience for high school
students.
1.1. Purpose of the System
The Luminary Project improves the learning process of a high school student with a personalized, AI-driven
educational platform. The system will optimize the learning curve by identifying the weak areas in which the
student might be having difficulties, thus generating targeted questions that will help the student channel
more effort to his or her weak points in order to improve his or her grade. This system also reduces
interactions with instructors, since it uses the automated generation of personally adaptive learning materials.
Another quintessential component of the Luminary platform is that it allows for interaction with the
educational content, providing a study environment for the student to access materials in a way close to that
of his or her style of learning. This includes the integration of user-generated content, where students can
submit their materials such as presentations, receive AI-generated answers, and interact with educational
tools that guide their study practices. Luminary does not only personalize education but goes further to
advance the education experience in making sure that students do not only learn but apply and comprehend
what they learn.
1.2. Design Goals
The project is aimed at providing a safe, secure, effective, and educational design for students.
• Scalability in Design
The design objective is to ensure that the system is designed to handle constant users and data load
and function without delays. The system should have the capability to handle a number of users and
data, maintaining the performance of this system as smooth as possible with the upsurge of users.
• Security
The system contains inbuilt security mechanisms for protecting data, ensuring confidentiality through
encryption, secure authentication, and adhering to data protection regulations.
• User-Friendliness
The platform should be user-friendly with an easy-to-navigate and intuitive interface for students,
teachers, and administrators, ensuring a seamless user experience.
• Integration
The design objective emphasizes freedom in interaction and ease of data interchange and
collaboration with existing learning tools and platforms.
• Adaptable Design
The system is designed for adaptability, enabling future enhancements and changes with minimal
rework and allowing for the integration of new features and technologies.
• Performance
The system should include faster response times and efficient data handling to provide a smooth and
user-focused interface.

The document for Luminary serves as an overview of the system's design, objectives, and architecture. The

2. Proposed Software Architecture
2.1. Overview
The proposed software architecture is designed to provide a comprehensive platform for high school students
preparing for university entrance exams. Leveraging modern technologies and best practices, the architecture
is modular, scalable, and efficient, ensuring optimal performance and user experience.
Microservices Architecture
The system adopts a microservices architecture, dividing the application into small, independent services,
each responsible for specific functionalities. The architectural design offers several advantages:
• Scalability: Each microservice can be scaled independently based on demand, ensuring optimal
resource utilization.
• Flexibility: Microservices enable rapid development and deployment of new features, as changes can
be made to individual services without affecting the entire system.
• Fault Isolation: Failures in one microservice do not impact the entire system, enhancing fault
tolerance and reliability.
• Technology Diversity: Different microservices can be developed using different technologies,
allowing teams to choose the most suitable tools for each task.
Communication and Integration
Communication between microservices is facilitated through well-defined interfaces and message queues.
Apache Kafka is utilized as a message broker to enable asynchronous communication and decouple services.
This approach enhances scalability, fault tolerance, and overall system performance.
Containerization with Docker
To streamline deployment and ensure consistency across different environments, the system leverages
containerization with Docker. Each microservice is packaged into a lightweight, portable container along
with its dependencies, providing a consistent runtime environment. Docker enables easy deployment, scaling,
and management of microservices, contributing to improved agility and reliability of the system.

The Luminary project is going to be a web application. In that application, we are going to use different
technologies.
Frontend: Using Angular.js to provide a responsive and interactive user interface.
Backend: Implemented in Java (Spring Boot) and Python.
Databases: PostgreSQL is used for structured data (e.g., user information, performance metrics) while

Messaging and Asynchronous Processing: Kafka is used as the message broker to facilitate asynchronous
communication between microservices.

2.4. Persistent Data Management
Data management in the Luminary platform involves multiple storage solutions to handle different types of
data efficiently:
Structured Data: Stored in PostgreSQL, which manages relational data such as user profiles, user-uploaded
questions (classified as course and subject), and task assignments.
Unstructured Data: Stored in MongoDB, which handles documents, chat logs, and AI-generated content.
MongoDB’s flexible schema design is well-suited for the varied and dynamic nature of this data.
Backup and Recovery: Regular backups of both PostgreSQL and MongoDB databases are scheduled to
prevent data loss.
2.5. Access control and security
User Authentication
Our users have to pass the authentication step to reach the platform. Users can be logging in to the
system using their email and password. These emails and passwords will be encrypted in our
databases.
Also, they can be logging in to our system just using their Gmail. In this case, Google's security
system handles the authentication.
Data Encryption
To be able to use some parts of the our platform users have to upload their documents into our
system. These documents will be stored in the database and will be encrypted.
2.6. Global software control
In the Luminary platform global software control is managed through service coordination. This control
system ensures that all the small parts of the system (microservices) work together smoothly.
2.7. Boundary conditions
Boundary conditions for the Luminary platform include:
• System Initialization
System Initialization prepares the system to start up correctly. We are going to use automated scripts
to set up databases, deploy services, and configure settings, ensuring everything is ready to go.
• Shutdown Procedures
Shutdown Procedures safely stop the system when needed. To stop the system we make sure all active
sessions are completed and data is saved before stopping the services.
• Disaster Recovery
Disaster Recovery restores the system after a major failure. Our databases will be backed up at
regular intervals to avoid data loss in unexpected situations.

3. Subsystem services
3.1. Login System
The Login System serves as the gateway for users to access the Luminary platform securely. It provides
authentication mechanisms to verify user identities before granting access to the system. Users can log in
with their credentials, including username and password. In case of invalid credentials or non-existent
accounts, the system prompts users to sign up for an account. Additionally, the system offers a "Forgot
Password" functionality, allowing users to reset their passwords securely via email. Upon successful
password reset, users are directed back to the login screen to log in with their new credentials. The Login
System ensures a seamless and secure authentication process, laying the foundation for user interaction with
Luminary's various features and subsystems.
3.2. MyProfile System
The MyProfile System empowers users to manage their personal information, preferences, and settings
within the Luminary platform. Upon accessing the MyProfile Component, users are presented with a
clickable profile button, leading them to a comprehensive view of their profile details. Here, users can
interact with various actions related to their profile, such as updating credentials, setting preferences, and
toggling between light and dark mode. Additionally, the system provides a logout button for users to securely
sign out of their accounts. The MyProfile System offers a user-centric approach, allowing individuals to
customize their Luminary experience according to their preferences and needs.
3.3. Dashboard
The Dashboard serves as the central hub of the Luminary platform, providing users with an intuitive interface
to navigate and access key features and functionalities. Upon accessing the dashboard, users are presented
with essential components tailored for student use, including Analyze, Daily Study, History, MyTasks, and
Map. Each component is designed to facilitate efficient navigation and interaction, adhering to UI/UX
principles for optimal user experience. The Dashboard acts as a gateway to Luminary's diverse features,
enabling seamless access to educational resources, study tools, and performance insights.
3.4. Analyze System
The Analyze System offers comprehensive tools and functionalities to assess and enhance student
performance in preparation for university entrance exams. Users can upload wrong questions, select the
number and difficulty level of practice questions, and visualize areas of weakness through charts or graphs.
The system generates personalized practice questions based on user preferences and directs users to solve
them in dedicated practice sessions. After completing practice sessions, users receive feedback on their
success rates and are prompted with next steps, fostering continuous improvement and learning.

3.5. Daily Study System
The Daily Study System provides students with a structured environment for daily study routines and
academic exploration. Users can upload documents or access MEB books for study purposes. The system
applies page number checks and facilitates seamless transitions between different study environments, such
as Smart Study and MEB Check-In. Through the Daily Study System, students can efficiently organize their
study materials, engage in focused study sessions, and access AI-powered tools for document summarization
and question answering.
3.5.1. Smart Study Environment
The Smart Study Environment offers a dynamic and interactive space where students can enhance their study
experience through advanced AI functionalities. Within this environment, students can engage with a chatbot-
like interface to request document summarization, making lengthy study materials more digestible.
Additionally, students can ask questions related to their uploaded documents, receiving AI-supported answers
that leverage large language models to provide accurate and contextually relevant responses. This
personalized study assistant helps students focus on key information and clarifies complex topics, facilitating
a deeper understanding and more efficient study sessions.
3.5.2. MEB Check-In Environment
The MEB Check-In Environment is designed to support students by providing easy access to official MEB
books and the educational content within them. Through a user-friendly chatbot interface, students can enter
topics they have questions about. The system scans the MEB books using advanced AI technologies to locate
the desired topics and provides detailed explanations from the relevant sections of the books. This
environment ensures that students can find comprehensive and structured information aligned with the
curriculum, enhancing their learning experience and supporting their academic needs across different grade
levels.
3.6. History System
The History System enables users to monitor their study progress and track past activities within the
Luminary platform. Users can review previously uploaded questions, categorize them based on courses, and
classify question topic titles for easy reference. By providing insights into past performance and study
patterns, the History System empowers users to identify strengths and areas for improvement, facilitating
informed decision-making and personalized learning strategies.

3.7. MyTasks System
The MyTasks System offers users a centralized platform to manage and track their academic tasks and
assignments. Users can view tasks categorized into "To-Do," "In-Progress," and "Done" states, assign tasks
to themselves, and add descriptions for clarity. The system supports drag-and-drop functionality for seamless
task management and provides a user-friendly interface for efficient task organization and tracking. With the
MyTasks System, users can stay organized, prioritize tasks, and stay on top of their academic responsibilities
effectively.
3.8. Map System
The Map System provides users with a visual representation of educational challenges across different
regions of Turkey. Users can interact with a clickable and hoverable map, where cities are colored based on
the courses that are most challenging in each region. By clicking on a city, users can access details regarding
the challenging courses in that specific location, including percentages and insights. The Map System offers
valuable insights into regional educational trends and facilitates targeted interventions and support initiatives
to address academic challenges effectively.

5. Work Plan
The Luminary Project aims to develop an educational platform aimed at optimizing the learning experience
for high school students. The work plan outlines the phases to achieve the project's objectives, using an Agile
software development approach to ensure flexibility and iterative progress. Our team has completed the
Planning and Design phases, and the details of these phases are explained.
The planning phase of the Luminary Project focused on identifying and documenting the educational needs
and system requirements through a comprehensive needs assessment. We conducted several meetings with
key stakeholders, which are students, to gather detailed insights into their needs and expectations. This
collaborative approach ensured that we captured a broad range of requirements and prioritized them based on
their importance and urgency. The outcome of the process was a detailed requirements specification that laid
the foundation for the subsequent phases of the project. Then, we evaluated various software development
methodologies to determine the best approach for the project. Given the dynamic nature of the requirements
and the need for flexibility, we decided to adopt the Agile process. Agile's iterative approach allows us to
accommodate changes and continuously improve the system based on user feedback. Following this decision,
we broke down the project into smaller, manageable tasks and created a comprehensive project backlog. Each
task was prioritized and assigned to different sprints, ensuring a clear and structured path for the development
phase.
In the design phase of the Luminary Project, our focus was to develop a high-level system design that
encompassed the overall architecture and key components of the Luminary platform. We started by defining
the system architecture, which includes critical components such as user management, AI engine, data
analysis, user interface, integration layer, and security. To represent the system's structure and behavior, we
created UML diagrams, including class diagrams, sequence diagrams, and use case diagrams. These diagrams
provided a visual representation of the system, aiding in better understanding and communication among
team members. An essential part of the design phase was the creation of detailed user interface prototypes.
Using Figma, a powerful design tool, we developed mock-ups of the user interface to ensure it was intuitive
and user-friendly. These prototypes were subjected to usability testing with a small group of users, whose
feedback was invaluable in refining the designs. The iterative process of testing and refining the prototypes
allowed us to finalize a design that meets the users' needs and expectations effectively. Furthermore, we
ensured that the design addressed key aspects such as scalability, security, user-friendliness, integration,
adaptability, and performance. The design phase concluded with the preparation of detailed mock-ups and
design specifications, ready for the development phase. By leveraging Figma and conducting thorough
usability testing, we ensured that the final design is both functional and aesthetically pleasing, setting a
strong foundation for the development phase.
